# SauceDemo UI Automation Framework

Framework для автоматизации UI-тестирования приложения [Swag Labs](https://www.saucedemo.com/) с использованием **Python**, **Pytest**, **Allure** и **Selenium**.

## Обзор проекта

- Чистая и масштабируемая архитектура: **POM + Components + Elements**
- Поддержка нескольких браузеров
- Конфигурация через **Pydantic Settings** и **.env** файлы
- Поддержка запуска на различных окружениях без изменения кода
- **Data-driven** тестирование через **Pytest Fixtures** и **фабрики тестовых данных**
- Переиспользование состояния авторизации через **cookies**
- Отчетность с **Allure** 
- **CI/CD** через GitHub Actions + автоматическая публикация **Allure-отчета** на **GitHub Pages**
- Параллельный запуск и рераны
## Основная идея
Вместо «толстых» Page Object, разделение на:
- **Elements** - UI-элементы (Dropdown, Input и т.д.) с строго определённым поведением
- **Components** - переиспользуемые UI-блоки (карточка товара, navbar)
- **Pages**  
  - Собирает вместе компоненты и элементы
  - Делает тесты максимально читаемыми и близкими к бизнес-сценариям

## Преимущества подхода
- **Типобезопасность и подсказки IDE**  
Каждый элемент имеет строго заданные действия (например, Button → click, Input → fill), что снижает количество ошибок на этапе написания кода
- **Переиспользование компонентов**  
UI-компоненты (карточка товара, navbar) используются на разных страницах, что уменьшает дублирование кода
- **Упрощение поддержки**  
Изменения UI вносятся в одном месте (элемент или компонент) и автоматически применяются во всех тестах
- **Централизованное логирование и Allure-отчётность**  
Все базовые действия элементов (click, fill, check_visible и др.) уже содержат allure.step и логирование. Благодаря этому тесты автоматически получают подробные шаги отчёта и логи без дублирования кода в каждом тесте.
- **Масштабируемость**  
Архитектура легко расширяется при росте приложения за счёт добавления новых страниц и компонентов без усложнения структуры
- **Повышение читаемости тестов**  
Тесты выглядят как пользовательские сценарии, а не как наборы кликов по локаторам

## Начало работы

### Клонирование репозитория

Для начала склонируйте репозиторий проекта с помощью Git:

```bash
git clone https://github.com/Vladimir-KNHV/saucedemo-selenium.git
cd saucedemo-selenium
```

### Создание виртуального окружения

Рекомендуется использовать виртуальное окружение для управления зависимостями проекта. Следуйте инструкциям для вашей операционной системы.

#### Linux / MacOS

```bash
python3 -m venv venv
source venv/bin/activate
```

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### Установка зависимостей

После активации виртуального окружения установите зависимости проекта из файла `requirements.txt`:

```bash
pip install -r requirements.txt
```



### Запуск тестов с генерацией Allure-отчета

Для запуска тестов и генерации Allure-отчета используйте команду:

```bash
pytest -m "regression" --alluredir=./allure-results
```

Команда выполнит все тесты проекта и выведет результаты в терминал.

### Просмотр Allure-отчета

После выполнения тестов вы можете сгенерировать и открыть Allure-отчет с помощью команды:

```bash
allure serve allure-results
```

После выполнения команды отчет Allure автоматически откроется в браузере по умолчанию.
